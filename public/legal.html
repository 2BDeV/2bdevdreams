Cookie Protocol

Data persistence and tracking parameters.

1. What are Cookies?

Cookies (and Local Storage) are small text files or data packets stored on your device when you visit our site. We use them to make the system work efficiently, remember your preferences (like closing this popup), and analyze how the site is performing.

2. Technologies We Use

We use two main types of local data storage:

  • Browser Cookies: Small files sent from our server to your browser.
  • Local Storage: A modern way to store data directly in your browser without sending it to the server every time.

System Data (Local Storage)

These are strictly necessary for the website's UI logic to function.

Data Key Purpose Duration
cookie_consent Stores your choice (accepted/declined) so the cookie banner doesn't reappear. Persistent
sys_msg_[ID] Remembers if you have closed a specific "Retro System Announcement" popup. Persistent
isAdmin Grants access to maintenance mode if you are the site owner. Session

Third-Party Services

We use trusted external providers who may set cookies to ensure security and performance.

Provider Purpose
Cloudflare (Turnstile) Security. Checks if you are a human or a robot when sending messages. This is essential for preventing spam.
Vercel Analytics Performance. Collects anonymous data about page load speeds and visitor numbers to help us improve the site.

3. Managing Your Preferences

Since we primarily use Local Storage for functionality, you can clear this data at any time by clearing your browser's "Browsing Data" or "Site Data".

If you declined cookies in our banner, we will not run tracking analytics scripts, but "Essential" data (like the security check for the contact form) will still function to keep the site secure.
